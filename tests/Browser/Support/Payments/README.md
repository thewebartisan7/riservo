# Payments Browser Support

P0 provides the shared browser-test foundation for the parked E2E-PAYMENTS roadmap. All Stripe calls must go through `Tests\Support\Billing\FakeStripeClient`; browser mode returns hosted-Stripe stand-ins under `https://stripe.test/external/{uuid}`.

## PaymentsTestCase

`tests/Browser/Payments/*` is bound to `PaymentsTestCase` from `tests/Pest.php`. The base class refreshes the database, registers `FakeStripeClient::forBrowser($this)`, exposes `$this->paymentsWorld()`, and signs Connect webhook POSTs.

```php
$this->dispatchStripeConnectEvent('checkout.session.completed', [
    'account' => $world['connectedAccount']->stripe_account_id,
    'client_reference_id' => (string) $booking->id,
    'id' => $booking->stripe_checkout_session_id,
]);
```

## PaymentsWorld

Use the fluent builder for the default payment graph: onboarded Business, admin, service, provider, seed customer, and optional connected account.

```php
$world = $this->paymentsWorld()
    ->withActiveStripeAccount()
    ->withOnlinePaymentMode()
    ->build();
```

## ConnectedAccountPage

Wraps `/dashboard/settings/connected-account`. It asserts D-185 by checking requirement counts and CTAs, not raw Stripe requirement paths.

```php
ConnectedAccountPage::openAs($world['admin'])
    ->shouldShowActiveState($world['connectedAccount']);
```

## BookingPage

Wraps the public booking funnel and delegates month-aware slot selection to `BookingFlowHelper`.

```php
BookingPage::open($world['business'])
    ->chooseService($world['service'])
    ->chooseAnySpecialist()
    ->chooseDateAndTime($targetDate)
    ->fillCustomerDetails($details)
    ->continueToSummary();
```

## BookingSummaryStep

Wraps the review/payment step. Error assertions target the `useHttp` validation rendering (`http.errors.{discriminator}`), not custom local toasts.

```php
$summary->shouldShowTwintBadge()
    ->confirm()
    ->shouldShowUseHttpError('This slot is no longer available.');
```

## PaymentSuccessPage

Wraps `/bookings/{token}/payment-success?session_id=...`.

```php
PaymentSuccessPage::open($booking, $booking->stripe_checkout_session_id)
    ->shouldShowProcessingState()
    ->followBookingStatusLink();
```

## BookingManagementPage

Wraps `/bookings/{token}` for customer-facing booking management.

```php
BookingManagementPage::open($booking)
    ->shouldShowBookingDetails()
    ->shouldShowPaymentState('Paid');
```

## DashboardBookingDetail

Wraps the admin booking-detail sheet. D-184 assertions use the server-side redirect endpoint paths generated by the same route contract as the Wayfinder helpers in `@/actions/App/Http/Controllers/Dashboard/StripeDashboardLinkController`; raw Stripe IDs should not appear in page source.

```php
$detail = new DashboardBookingDetail($page);

$detail->shouldShowPaymentDeeplinkFor($booking)
    ->shouldShowDisputeDeeplinkFor($booking)
    ->shouldNotExposeRawStripeIds('ch_test_raw', 'dp_test_raw');

$detail->refundsPanel($booking)->shouldShowDeeplinkFor($refund);
```

## PayoutsPage

Wraps `/dashboard/payouts`. It asserts D-185 requirement counts, G-005 login-link errors from `http.errors.login_link`, and unsupported-market banners.

```php
PayoutsPage::openAs($world['admin'])
    ->shouldShowRequirementsCount(2)
    ->shouldShowLoginLinkError('Could not open Stripe right now.');
```

## SettingsBookingPage

Wraps `/dashboard/settings/booking`. D-183 is intentional: `payments.supported_countries` changes gate state only; the MVP explanatory copy remains CH-centric until the locale-list audit.

```php
SettingsBookingPage::openAs($world['admin'])
    ->shouldShowPaymentModeControls()
    ->shouldShowChCentricUnsupportedCopy();
```
